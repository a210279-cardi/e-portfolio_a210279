Project 2 · Advanced

ScanBuddy — Connected, Persistent & Sensor-Aware

Problem Statement: Project 1 required manual ingredient entry. Real users need instant, automatic results. Project 2 upgrades ScanBuddy with a live barcode scanner (camera sensor), the OpenFoodFacts REST API for automatic ingredient lookup, Room Database for offline product history, and Firebase Firestore so users can share safe products with the community — directly contributing to SDG 3 by making health information accessible and shareable at scale.

Lessons Learned: Integrating four different technical pillars at once revealed how important threading is in Android. Retrofit calls, Room queries, and Firestore reads all need to run on background coroutines — blocking the main thread even briefly causes visible jank. I also learned that runtime permissions must be requested explicitly at the point of use, not just declared in the manifest. Firebase setup required careful SHA-1 fingerprint registration which cost debugging time but was a valuable real-world lesson.
04 · Reflection & Learning Journey

Growth from Lab 1 to Project 2

Every stage added a layer of complexity. This is how I grew through each one.

Lab 1 — First Composable 🧱

Writing my first @Composable function felt strange. I kept reaching for XML. By the end of the lab I understood why declarative UI is more predictable — the UI is always a pure function of state, which means fewer hidden bugs.

Lab 2–3 — State & Design 🎨

Managing state correctly was the trickiest conceptual shift. I initially hoisted state too low and had components not updating correctly. Material Design 3 helped me think about user experience as part of the code, not as an afterthought.

Lab 4 — Architecture Click 💡

This was the moment everything clicked. ViewModel + NavController together solved problems I had been working around: screen-to-screen data sharing, back-stack management, and surviving device rotation. I restructured my mental model of Android apps completely.

Project 1 — First Full App 📱

Building 5 complete screens for ScanBuddy taught me the gap between understanding concepts individually and combining them into a coherent product. UI feedback from the Live Q&A helped me improve navigation flow and screen transitions for Project 2.

Lab 5 — Data Persistence 💾

Room Database was initially intimidating — the annotation-based approach felt verbose. But once I saw data surviving app restarts, I appreciated the abstraction. Writing suspend functions in the DAO and collecting from StateFlow in the ViewModel felt natural.

Project 2 — Connected App 🌐

This was the hardest and most rewarding stage. Wiring Retrofit, Room, Firebase, and the camera sensor together forced me to think about error states, loading states, and offline behaviour simultaneously. The app I built can genuinely help real users — that makes the difficulty worth it. SDG 3 is not just a theme label in my app; it is the reason every feature exists.
